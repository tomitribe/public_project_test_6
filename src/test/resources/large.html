Tomcat users
REST services to manage the tomcat-users.xml file. This resource returns 3 different types:
  • UserInfo: this is the user as you can define in the tomcat-users.xml file.
    • username: the user name (key in the file)
    • name: the name of the user
    • user: the description of the user
    • roles: a set of group
    • groups: a set of role
  • GroupInfo: represents a group as per the tomcat-users.xml file corresponding tag.
    • groupname: the group name (key in the file)
    • name: the name of the group
    • description: the description of the group
    • roles: the set of roles belonging to this group
  • RoleInfo: represents a role of the tomcat-users.xml file.
    • rolename: the role name (key in the file)
    • name: the role name
    • description: the description of the role

application
This resource provides all management features to administrate all applications hosted if in the Apache TomEE server. This resource will respond with the following objects:
  • ContextInfo: object holding a set of information about the server
    • hostname: the name of the host the webapp is deployed against.
    • name: the name of the context (unique in the host)
    • stateName: the current state of the application (ie. STARTED, STOPPED, FAILED, etc)
    • info: technical information on the implementation of the Context
    • baseName: the baseName
    • displayName: the human readable name.
    • docBase: the document base (or context root) used for deployment.
    • path: the url path.
    • sessionTimeout: the session timeout defined for this application.
    • crossContext: shall this application access the request dispatcher for other web applications running on the same virtual host
    • reloadable: is the webapp reloadable when changes are detected in WEB-INF/lib or WEB-INF/classes?
    • webappVersion: the version of the webapp. Rarely used.
    • startTime: the number of milliseconds since it's been started.

server
This resource provides all management features to administrate the Apache TomEE server. This resource will respond with the following objects:
  • ServerInfo: object holding a set of information about the server
    • name: the name of the server configured in the server.xml file
    • base: the ${catalina.base} system property
    • version: the current version of the server
    • status: the status of the server
    • updtime: the amount of ms the server is running
    • connectors: list of declared connectors in the server.xml file
    • systemProperties: all system properties. No magic there.
    • vmName: JVM name
    • vmVendor: JVM vendor
    • vmVersion: JVM version
    • serverInfo: server information
    • shutdown: shutdown command
    • port: shutdown port
  • HostInfo: represents one host configured in the server.xml file.
    • name: the name of the host
    • appBase: the application base directory for the virtual host
    • autoDeploy: should the server periodically scans for new or updated applications while running
    • deployOnStartup: should the applications of this host be auto deployed at startup
    • hostInfo: the host class name and information
    • stateName: current state of this host
    • realmInfo: realm configured in this host (only the top level currently)
    • contextPaths: list of applications belonging to this host

sessions
HTTP Session management. This resource will respond with the following objects:
  • SessionInfo: object holding a set of information about the session
    • hostname: the host name the application belongs to
    • contextname: the application context name (unique in the host scope)
    • id: formaly the jsessionid
    • creationTime: the creation time of the session
    • lastAccessedTime: the last time the server served a request for that user
    • userTime: the amount of time the user has been active
    • inactiveTime: the amount of time since the server did not served any request from that session
    • ttl: Time To Live before the session expires.
